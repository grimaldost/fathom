Installed skills and when each applies. Before starting substantive work, consider
whether any of these fits the request; load the best fit with the Skill tool. Naming a
skill is not required -- 'nothing fits, proceeding directly' is a valid outcome.

- humblepowers:choosing-tools -- deciding which installed skill, if any, owns a task.
- humblepowers:systematic-debugging -- a reported defect; find and fix the underlying
  cause rather than the reported symptom, and check for other sites the same cause hits.
  Not for: writing new features from scratch.
- humblepowers:test-driven-development -- writing new production code against a test seen
  failing first (red-green-refactor).
- humblepowers:verification-before-completion -- proving a change works before calling it
  done; leave a check that would fail if the fix regressed. Not for: the initial design.
- humblepowers:brainstorming -- turning a vague or bundled request into an agreed design
  before implementation.
- humblepowers:receiving-code-review -- evaluating incoming review feedback on its merits.
- humblepowers:planned-execution -- a midweight change wanting an executable plan plus a
  per-task review loop.
- humblepowers:skill-authoring -- writing or revising a skill's own definition.
- humblepowers:choosing-models -- routing a sub-task to a model tier and effort level.
- engineering-discipline:data-engineering-discipline -- data or pipeline correctness;
  verify the actual output against the input rather than infer from the code; watch for
  quiet row drops, duplication, and boundary miscounts. Not for: pure UI or config edits.
- engineering-discipline:python-engineering -- Python project architecture, packaging,
  and tooling (src layout, uv, ruff, typing).
- session-workflow:context-handoff -- authoring a paste-ready brief to hand work to a
  fresh session.
- session-workflow:review-panel -- a blind fresh-eyes review of a design before a
  hard-to-reverse call.
- session-workflow:toolkit-awareness -- taking inventory of the installed toolkit.
- session-workflow:journaling-sessions -- capturing session knowledge into entries.
- session-workflow:consolidate-knowledge -- distilling captured entries into durable guidance.
- session-workflow:compaction-survival -- keeping a control anchor across long autonomous runs.
- session-workflow:tool-feedback -- writing a per-session dogfooding report for a tool.
- session-workflow:feedback-triage -- turning a feedback corpus into an improvement backlog.
