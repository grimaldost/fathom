# Update Strategy

## How References Stay Current

The reference files in this skill are generated from the installed `treasuryutils`
package using `scripts/generate_references.py`. Each generated file includes a
version stamp at the top indicating the library version it was built against.

The nine **generated** mirrors are the eight per-domain API files
(`authenticator_api.md`, `calendartools_api.md`, `capitaltools_api.md`,
`compute_common_api.md`, `datatools_api.md`, `equitytools_api.md`,
`financialtools_api.md`, `quanttools_api.md`) plus `yaml_contracts.md`. The
remaining reference files (`decision_matrix.md` and this `update_strategy.md`)
are hand-written and are **not** produced by the generator.

---

## Freshness gate (`--check`)

`generate_references.py --check` is the canonical freshness gate. It regenerates
all nine mirrors **in memory**, strips the two volatile stamp lines
(`- treasuryutils_version:` and `- generated_at_utc:`) from both the freshly
generated text and the committed file, and byte-compares the remainder. It writes
nothing and exits:

- `0` ŌĆö all nine mirrors are current;
- `1` ŌĆö one or more mirrors drifted (each drifted file is named on stderr), or
  fewer than nine mirrors could be generated and compared.

Run it manually with:

```bash
uv run python plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/generate_references.py \
  --check
```

The gate is wired into the test suite at
`tests/test_consumer_references.py::test_generated_references_fresh`, which runs
`--check` in the suite interpreter ŌĆö so CI fails if the committed mirrors fall out
of date ŌĆö and asserts the run writes nothing.
`test_check_detects_planted_drift` proves the gate is non-vacuous (a tampered
mirror is detected).

### What `--check` does and does not prove

`--check` is a **tamper/consistency** gate, not a freshness gate. It regenerates
in memory against **whatever `treasuryutils` is installed in the environment
running it** and byte-compares against the committed mirrors. So:

- **It proves:** the committed mirrors match the generator's output for *this
  environment's* installed `treasuryutils` ŌĆö no hand-edit, no missing file, no
  drop below the expected mirror count.
- **It does NOT prove:** the mirrors match the *published release*. If the
  generation environment resolves a stale or dev version (e.g. an editable
  checkout stamped `1.5.2.dev` while the release is `1.8.0`), `--check` is
  self-consistent and passes GREEN on stale references. That is exactly the
  2026-07-10 incident.

Release freshness is `--require-version`'s job (Bump Procedure step 5), not
`--check`'s. The two compose: `--check --require-version <X>` gates both tamper
and release-currency at once.

## Bump Procedure

When `treasuryutils` is updated to a new version:

1. **Install the new version** in the treasuryutils dev environment:
   ```bash
   uv sync --all-extras
   ```
   (`--all-extras` is the right regen command. Note the `equity` and `capital`
   extras are intentionally empty placeholders, so they pull no deps; the
   `equitytools`/`capitaltools` mirrors are generated against the
   `datatools,quant-math,quant-optimizer` extras instead ŌĆö see `DOMAIN_EXTRAS`
   in `scripts/generate_references.py`.)

2. **Regenerate all references**:
   ```bash
   uv run python plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/generate_references.py \
     --output-dir plugins/treasuryutils-consumer/skills/treasuryutils-usage/references/
   ```
   (`--output-dir` defaults to that references directory, so the flag is optional.)

3. **Review changes** ŌĆö check the generated diffs for new/removed/renamed symbols.
   Most of a refresh is mechanical; the API-shape changes are what to read.

4. **Commit** the regenerated references ŌĆö all nine mirrors plus
   `.generated_meta.json`. `yaml_contracts.md` is generated by this same script
   (from the DataTools config Pydantic models) ŌĆö do **not** hand-edit it;
   regenerate instead.

5. **Confirm the gate is green AND stamped against the release** ŌĆö run `--check`
   together with the release gate, substituting the version being cut for `<new
   version>` (e.g. `1.8.0`):
   ```bash
   uv run python plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/generate_references.py \
     --check --require-version <new version>
   ```
   `--require-version` fails (exit `4`) unless the resolved `treasuryutils` is a
   *published release* of exactly `<new version>` ŌĆö the base version matches and it
   is not a dev release (a `+local` segment is allowed). It composes with `--check`,
   so this one command is the release-cut gate: it blocks a cut that would ship
   references stamped against a stale or dev version (the 2026-07-10 incident, where
   `1.5.2.dev` mirrors reached a consumer running `1.8.0`). If it fails, the message
   names the resolved vs required version and the two usual causes (stale
   checkout/venv ŌåÆ `uv sync --all-extras`; unreachable git tags under hatch-vcs ŌåÆ
   `git fetch --tags`).

## Consumer Installation

Consumer projects get this skill as a Claude Code plugin from the treasuryutils
marketplace:

```text
/plugin marketplace add stone-payments/treasuryutils
/plugin install treasuryutils-consumer
```

To update, refresh the marketplace and re-run `/plugin install
treasuryutils-consumer` to pull the latest published references.

## CI Automation (optional)

A GitHub Actions workflow can automate the bump process:

```yaml
name: Bump treasuryutils skill references
on:
  schedule:
    - cron: '0 8 * * 1'  # Weekly on Monday at 08:00 UTC
  workflow_dispatch:

jobs:
  bump:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4

      - name: Install dependencies
        run: uv sync --all-extras

      - name: Regenerate references
        run: |
          uv run python plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/generate_references.py \
            --output-dir plugins/treasuryutils-consumer/skills/treasuryutils-usage/references/

      - name: Freshness gate + lint
        run: |
          uv run python plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/generate_references.py --check
          uv run ruff check plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/
          uv run ruff format --check plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/

      - name: Create PR if changed
        run: |
          git diff --quiet || {
            git checkout -b chore/bump-treasuryutils-skill-$(date +%Y%m%d)
            git add plugins/treasuryutils-consumer/skills/treasuryutils-usage/references/
            git commit -m "chore: bump treasuryutils-usage skill references"
            git push origin HEAD
            gh pr create --title "chore: bump treasuryutils skill references" \
              --body "Automated weekly refresh of treasuryutils-usage skill references."
          }
```

## Staleness (advisory)

The generator also supports a time-based staleness check. This is **advisory** ŌĆö
the content `--check` above is the binding freshness gate ŌĆö but it is useful for a
quick "how old are these?" signal:

```bash
uv run python plugins/treasuryutils-consumer/skills/treasuryutils-usage/scripts/generate_references.py \
  --check-staleness-only --stale-after-hours 168
```

This exits with code 3 if the references are older than the specified threshold,
based on the `generated_at_utc` timestamp recorded in `.generated_meta.json`.
