# LLM signature — trailer spec (llm-signature/v1)

The signature is a pair of [git trailers](https://git-scm.com/docs/git-interpret-trailers)
recording the provenance of agent-assisted work. It is generated by
`scripts/render_signature.py` from live sources; a hand-typed signature is a
spec violation (it rots, and wrong provenance is worse than none). It is a
provenance *record*, not a cryptographic signature — there is no integrity
mechanism, and none is claimed.

## Grammar

```text
Assisted-By: <model-id>
Agent-Stack: <item>[; <item>]...

item     := <name>[@<version>][ (<marketplace>)]
model-id := the exact model identifier as the transcript records it
```

Real model identifiers are often dated (e.g. `claude-sonnet-4-5-20250929`),
not the short aliases used in examples here — write read-back greps against
what your history actually contains. One line per trailer (no continuations),
emitted as the final paragraph of the commit message so git parses them as one
trailer block. Example:

```text
feat(pipeline): add schema-drift gate

Assisted-By: claude-sonnet-5
Agent-Stack: claude-code@2.1.211; engineering-discipline@2.1.0 (craft-collection); session-workflow@0.15.0 (craft-collection)
```

## Semantics

- **`Assisted-By` is singular and carries responsibility.** It names the model
  that wrote and orchestrated the change — the one accountable for it. Models a
  session merely delegated to (subagents, sidechains) are not listed; if the
  orchestrating model changed mid-session, the one at commit time signs. The
  model never appears as a `Co-Authored-By`, author, or committer.
- **`Agent-Stack` is environment-at-commit provenance.** Harness first
  (`claude-code@<version>`), then enabled plugins sorted by name. It records
  what was installed and enabled when the work was committed — a lockfile-style
  snapshot — not a claim that every listed plugin fired. Disabled plugins are
  excluded; `--plugin` narrows further when only part of the stack shaped the
  work. If nothing resolves, the trailer is omitted — an empty stack line
  carries no information. Known limits: the CLI reports installed global
  state, so `--plugin-dir` dev-session plugins are not represented, and a
  plugin updated mid-session reports its current version.
- **The marketplace label is a reference, not a URL.** `(craft-collection)`
  resolves via `claude plugin marketplace list` or the adopting repo's
  README/resolution table. Commits carry short stable keys; the repo carries
  the mapping to marketplace pages.
- **The scrub is anchored to the vendor's identity, not to a name.** Only
  flush-left `Co-Authored-By` lines carrying an `@anthropic.com` address or a
  claude+noreply bot address are removed, plus flush-left
  "Generated with … Claude" badge lines. A human co-author named Claude with
  their own email, indented or quoted examples, and body prose that mentions
  the vendor all survive. A `Claude-Session:` trailer is deliberately left
  untouched: it is session traceability, not attribution marketing, and some
  harnesses mandate it.

### Alternatives considered (and why they lost)

- **Repeated `Agent-Stack:` lines, one per item** (git-native multiplicity, as
  `Co-Authored-By` scales): rejected for v1 — the stack is one atomic fact
  about the commit, a single value diffs and greps as one line, and a
  multi-line block doubles the trailer footprint on every commit. The
  semicolon list is still mechanically splittable (`split('; ')`). Revisiting
  this is a `v2` change.
- **Augmenting `Co-Authored-By` instead of replacing it**: rejected by design
  policy — the owner's position is that an AI co-author line carries no
  information and pollutes authorship semantics (the model is not a
  contributor). Interop with tooling that counts AI co-authors is knowingly
  given up; `Assisted-By` is the queryable replacement.

## Resolution sources (why it is never hand-typed)

| Field | Source | Notes |
|-------|--------|-------|
| model | session transcript, last main-loop `assistant` message | `--model` / `--transcript` override; auto-discovery below |
| harness | `claude --version` | omit with `--no-harness` |
| plugins | `claude plugin list --json` | enabled only; `id` = `plugin@marketplace` |

**Auto-discovery is verified, and best-effort by design.** A live session
(`CLAUDECODE` set) is required; candidate transcripts are nominated by
munged-cwd directory match (this cwd, its parents, and its descendants), and a
candidate signs only if it was touched within the freshness window (30 min)
AND the cwd recorded inside it verifies against this one by exact path —
lossy-munging collisions (`my.repo` vs `my-repo`) cannot pass. When nothing
verifies, the script refuses to sign rather than guess. Residual limit, stated
honestly: two concurrent sessions in the *same* directory race on
newest-mtime; pass `--transcript` (hook payloads carry `transcript_path`) or
`--model` when determinism matters.

Failure is loud in render mode (exit 1, nothing printed) and safe in `--apply`
mode (scrub-only or untouched, exit 0 — including on non-UTF-8 messages, which
round-trip byte-exact): a signing problem must never invent provenance or
block a commit.

## Reading signatures back

```bash
git log --format='%h %(trailers:key=Assisted-By,valueonly)'          # who signed what
git log --grep='Agent-Stack: .*session-workflow@0\.15' --oneline     # work shaped by a given version
git interpret-trailers --parse < msg                                  # machine parsing
```

## Adopting in a repo

Adoption is one declaration in the repo's `CLAUDE.md` (or equivalent agent
instructions), e.g.:

```text
Sign agent-assisted commits per llm-signature (session-workflow): render the
trailers with the skill's script and append them verbatim; never add an AI
Co-Authored-By line.
```

That line is what the skill's "a project that adopts the signature" trigger
refers to. Optionally add a resolution table to the README mapping marketplace
labels to their pages, and/or wire the git hook below.

## Optional git hook (full mechanical enforcement)

`--apply` is idempotent (refreshes, never duplicates) and commit-safe: it
honors the configured `core.commentChar`, freezes everything after a scissors
line (`git commit --verbose`), and exits 0 on any failure. The recipe guards
its own preconditions so a missing `uv` or a moved plugin can never abort a
commit either. `${CLAUDE_PLUGIN_ROOT}` does not exist in git's environment —
substitute the installed plugin path (shown by `claude plugin list` as
`installPath`):

```bash
# .git/hooks/prepare-commit-msg  (chmod +x)
#!/bin/sh
# Signing is best-effort by design: a missing uv, script, or model must never
# block a commit.
SCRIPT="<installPath>/skills/llm-signature/scripts/render_signature.py"
command -v uv >/dev/null 2>&1 || exit 0
[ -f "$SCRIPT" ] || exit 0
uv run --no-project -- python "$SCRIPT" --apply "$1" || true
```

Outside a live agent session the hook scrubs boilerplate but signs nothing
(no `CLAUDECODE` in the environment — and even inside one, a transcript that
fails the freshness or cwd verification does not sign). Human commits
therefore pass through unsigned, which is the intended behavior.

## Versioning

This is `llm-signature/v1`. Additive fields (e.g. MCP servers in
`Agent-Stack`) may arrive without a version bump; a change to the meaning of
an existing trailer — including switching to repeated `Agent-Stack:` keys —
requires `v2` and a spec note here.
