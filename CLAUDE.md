# CLAUDE.md — fathom

fathom measures whether an AI coding tool is worth using: it runs real coding tasks under different
**execution strategies / tool configurations** (arms), scores results **blind to which arm produced
them**, and joins quality with economy (tokens, turns, wall-clock, USD) into longitudinal verdicts.
Scenario-blind tool-effectiveness evals. Stdlib-only core; built with the keel/convoy governed-series
method (`docs/method/`).

## Gate commands (all must pass before any commit)

```sh
uv run ruff format --check .
uv run ruff check .
uv run pytest                 # stdlib-runnable test suite (count moves as tests are added; don't gate on an exact N)
uv run fathom smoke             # real-spawn isolation gate — spends a little; run before any paid matrix
```

Core modules under `src/fathom/` and their `tests/test_*.py` import **stdlib only** and run without uv —
e.g. `python tests/test_scenario.py`. `uv` manages dev tooling (ruff, pytest) only; the spawning/ledger
core has zero third-party deps.

## Run an eval (the analysis)

An **analysis** = a scenario matrix run against a task **bank**, scored into a **scorecard**.

```sh
# 1. ALWAYS smoke first — proves spawn isolation + injection on REAL spawns before spending.
uv run fathom smoke                                              # expect "ALL PASS (8/8 checks)"

# 2. Plan first (spawns nothing): prints trial count + USD ceiling, and shows resume state.
uv run fathom run <bank> --dry-run [--repeats K] [--scenarios-dir DIR]

# 3. Run the matrix (real paid spawns). Resumable: re-invoking skips already-completed trials.
uv run fathom run <bank> [--repeats K] [--scenarios-dir DIR] [--limit N]

# 4. Render the scorecard from the ledger (idempotent — regenerate any time).
uv run fathom report <bank>
```

- **Banks** live in `tasks/<bank>/` (many now — `ls tasks/`). The reference skill bank is
  `skill-pyeng-v1` (`modernize-timeflow`; arms bare / generic-nudge / pyeng-skill, in
  `scenarios/skill-pyeng/`). The `series` arm (`scenarios/series.toml`) drives the **convoy** multi-PR
  engine per the engine-agnostic contract (`docs/specs/2026-07-03-series-engine-contract.md`). A sealed
  holdout is excluded by default; run it deliberately with `--include-holdout` (ADR-0005), which marks
  its trials holdout in the ledger.
- **⚠ `--scenarios-dir` is load-bearing for any bank that ships its own arms.** `fathom run` globs
  `<scenarios-dir>/*.toml` **non-recursively** (default `scenarios/`), so a bank with its own scenario
  subdir MUST pass it or the run silently uses the wrong arms:
  ```sh
  uv run fathom run skill-pyeng-v1 --scenarios-dir scenarios/skill-pyeng --repeats 3
  ```
- **Where results land:** `ledger/<bank>.jsonl` (append-only, **committed** — the longitudinal record)
  and `report/scorecard-<bank>.md` (**gitignored**, regenerated by `fathom report`). Read the scorecard's
  **Per-Criterion Pass Rates** table for the discriminating signal, not just the headline pass-rate.
- **Cost / rails:** ~$2/trial ceiling printed upfront (C4); a full v1 matrix is ~$20-40. `--dry-run` and
  `--limit N` on every entry; per-spawn budget cap; `fathom smoke` is the go/no-go gate before any paid
  matrix and at every resume.
- **Invalid runs are archived, never deleted** (append-only): `ledger/archive/`. A run truncated by a
  harness limit is re-run fresh and the old ledger archived (precedent: the run-1 invalidations).

## Architecture

Pipeline: `taskbank → scenario(resolve + config_hash) → Runner(claude-cli adapter) → StrategyExecutor →
grading(verifier-first, pairwise judge) → append-only ledger → report`. Full module map:
`docs/specs/2026-06-10-fathom-v1-design.md` §4; build detail + enforcement table:
`docs/specs/2026-06-10-fathom-v1-build.md`.

**Four load-bearing invariants** (each has an ADR under `docs/adr/`):
- **Blindness** (ADR-0003) — verifiers see only the final workspace (a stripped result-view + a
  whitelisted env); judges see outputs labeled A/B with scenario identity removed; economy joins
  **after** scoring. `src/fathom/grading/verifier.py`.
- **Append-only ledger** (ADR-0002) — no code rewrites a ledger line; reports regenerate from the
  ledger; resume key = `(bank, dataset_version, task_id, config_hash, repeat)`; only
  `status=="completed"` trials count as done. `src/fathom/ledger.py`.
- **Spawn isolation** (ADR-0004) — credential-only temp `CLAUDE_CONFIG_DIR`; headless **default-deny**
  (never `bypassPermissions` / `--dangerously-skip-permissions`); explicit allow/disallow lists.
  `src/fathom/adapters/claude_cli.py`. The ONE sanctioned non-adapter model call is the series engine in
  `src/fathom/strategies/series.py`.
- **Stdlib core** — modules under `src/fathom/` import stdlib only; `typing.Protocol` at the seams
  (`Runner`, `StrategyExecutor`).

## Authoring banks / scenarios (as-built schemas)

A scenario is **flat TOML** (NOT a `[scenario]` table — the design spec §4.3 example is stale; the
parser in `src/fathom/scenario.py` is the source of truth):

```toml
name = "pyeng-skill"          # arm name; "bare" is the special pairwise anchor in the report
adapter = "claude-cli"
model = "claude-opus-4-8"
strategy = "single-session"   # single-session | series | gated-session | gated-review | reprompt-session (unknown names are rejected)
effort = "high"

[tools]
source = "none"               # "none" | "repo"
allowed = ["Read", "Write", "Edit", "Glob", "Grep", "Bash(python:*)"]   # empty under default-deny == UNARMED
# disallowed = [...]          # optional belt-and-braces

[context]                     # optional — a treatment arm: this file's body is appended to the spawn's
inject = "assets/python-engineering.md"   # system prompt (--append-system-prompt-file). Path is relative
                              # to this scenario file; its CONTENT (sha256) enters config_hash.

[limits]
trial_timeout_s = 600
```

- `tasks/<bank>/bank.toml`: `name`, `dataset_version` (bump on ANY task/fixture/verifier change — it is
  in the resume key), `holdout = [...]`.
- `tasks/<bank>/<task-id>/task.toml`: `id`, `instruction`, `[limits] trial_timeout_s, max_turns`,
  `[verify] entry = "verify.py"` (optional `timeout_s`, default **60**, bounds the verifier
  subprocess — raise it for a `verify.py` that shells out to a heavier harness, e.g. a full pytest
  collect+run, or the verifier times out and the trial is scored an error). (`max_turns` IS plumbed
  to the spawn — set it high enough for the task's work, or the run truncates.)
- `verify.py`: reads **only** `argv[1]` (the result-view path), emits `{criterion: bool}` JSON to stdout,
  exits 0 iff the correctness gate holds. **No scenario identity in argv/env** (blindness). The schemas
  above are the source of truth (`src/fathom/scenario.py` and `taskbank.py` parsers).

## Conventions

Windows host — paths via `pathlib`, terminate process trees where you spawn. Never edit `ledger/` by hand
(append-only). Commit messages carry no AI-attribution trailer (GPG/SSH signing left as configured).
`report/`, `.pr-outputs/`, `.worktrees/`, `.venv/` are gitignored; `ledger/` is committed.
